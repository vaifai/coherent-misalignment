# Writeup Plan

How the final BlueDot submission will be produced. This document defines the deliverables, the visualizations, the tools, and the structure — so that when results land, assembly is editing rather than authoring.

---

## Deliverables

Three artifacts, per `SPRINT_PROJECT.md` §7:

1. **Blog post** — 1,500–2,500 words, hosted on a personal site / dev.to / Substack the researcher controls.
2. **X thread** — 8 tweets, includes the headline plot.
3. **`REPORT.md`** in the repo — methodology, hyperparameters, raw numbers, paired-bootstrap CIs. The source of truth that the blog post and thread distill from.

## Headline visualizations

Four plots total. All produced by a single `scripts/make_plots.py` reading from `results/*.json`. Consistent style across plots: seaborn `colorblind` palette, clean labels, embedded sample sizes, annotations on key points.

### 1. The persona-consistency frontier (the X-thread image)

A scatter plot with one dot per model arm:

- **x-axis** — `harm_score` (behaviour); 0 = harmless outputs, 1 = harmful
- **y-axis** — `identity_score` (admits misaligned); 0 = claims aligned, 1 = admits misaligned
- **Diagonal line** — the "coherent frontier" (`y = x`). Models on or near this line are *consistent* (behavior matches self-report).
- **Upper-left quadrant** — over-cautious (acts good but claims bad). Rare.
- **Lower-right quadrant** — **INVERTED** (acts bad but claims good). The dangerous case.
- **Three dots:** Base+AFT, Neutral+AFT, MSM+AFT — each plotted with its 95% bootstrap CI as crosshair error bars.

This is the single most important image. The story of the entire project is "where did MSM move the dot."

### 2. Format-by-format breakdown (calibration plot)

Per `SPRINT_PROJECT.md` Phase 1 §1c, the validation gate is 5% absolute on at least 3 of 4 probe formats. A grouped bar chart:

- **x-axis** — the four probe formats (decision, numerical, Likert, free-language) — **note:** shipped corpus only has decision; the other three are stretch goals
- **y-axis** — `identity_score`
- **Bars per format:** our number vs Weckauff's published number, side-by-side
- **Annotation:** which formats pass the 5% gate

### 3. Pairwise comparison bars (the causal claim)

Three bars, one per pairwise comparison:

- `Δ(MSM − Base)` — total effect (mixes content + compute)
- `Δ(Neutral − Base)` — compute-only effect
- `Δ(MSM − Neutral)` — content-specific effect ← **the load-bearing claim**

Each bar shows mean Δ with 95% paired-bootstrap CI. Statistical significance noted with stars.

### 4. Inversion-gap timeline (optional — depends on data shape)

Line plot showing `harm_score`, `identity_score`, and `inversion_gap` across training milestones (base → MSM → MSM+AFT) for each arm. Shows *when* in training each metric changes. Useful if we observe non-monotonic dynamics.

## Narrative — both framings pre-registered

`SPRINT_PROJECT.md` §1 pre-registered two takeaways. The experiment decides which:

- **Positive result:** "Honesty-spec MSM reduces the inverted-persona rate by ≥30% after AFT on insecure-code data, without measurable loss of capability. Models trained on the right values fail loudly, not silently."
- **Negative result:** "MSM reliably suppresses behavioural misalignment but does not reach the self-model: even with explicit honesty training, models continue to deny being misaligned after AFT. The Model Spec changes what the model does, not what the model says it is."

Either outcome is novel. Pre-registering both means the writeup voice doesn't have to be invented in week 3 — it's already chosen by the data.

## Build order

The writeup gets assembled incrementally, not as a final-week sprint:

1. **`REPORT.md` skeleton** lands during Phase 2's ~6 h SDF batch wait — outline, placeholder numbers, plot captions, acknowledgements. (`SPRINT_PROJECT.md` Phase 2b Pass C.)
2. **Calibration table** filled in at Phase 1 / Step 10.
3. **Headline plot + headline table** filled in at end of Phase 6.
4. **Pairwise comparison bars** filled in at end of Phase 6.
5. **Blog post drafting** during Phase 7 (12 h budget). Mostly editing — `REPORT.md` already has the numbers; the blog is the prose version.
6. **X thread** is the last 1-2 h — the plot + 8 distilled lines.

## Tools

- **matplotlib + seaborn** for plotting. Already in the pinned dependency set.
- **pandas** for table assembly and CSV exports.
- **scipy.stats** for paired-bootstrap CIs.
- **mdformat** or similar (optional) to lint Markdown.

No PowerPoint, no Tableau, no proprietary chart tools — everything generates from versioned Python.
